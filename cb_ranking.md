# 可转债三低排序说明

本文说明 `cb_main.py` 里当前“三低排序”的具体实现方式。

## 1. 基础过滤

在进入排序前，脚本会先做基础过滤，只排除这些不希望参与排序的转债：

- 正股名称含 `ST`
- 已公告强赎：`icons` 中包含 `O`
- 到期赎回：`icons` 中包含 `R`

对应代码：

- `cb_main.py` 中的 `get_cb_filter_reasons()`
- `cb_main.py` 中的 `filter_cb()`

也就是说，当前已经不再使用 `价格 <= 115` 作为筛选条件。

## 2. 三个排序维度

过滤后的转债，会按下面两个字段分别打分：

- 双低：`dblow`
- 规模：`curr_iss_amt`

两项规则完全一致，都是：

- 数值越低，排名越靠前
- 第 1 名得分最高
- 第 2 名得分次高
- 依次递减
- 当前实现与另一套服务端逻辑一致：即使数值相同，也按排序先后给不同 rank

例如：

- 某只债双低最低，则 `dblow_rank = 1`
- 某只债规模最小，则 `curr_iss_amt_rank = 1`

单个因子的得分公式是：

```text
score = total - rank + 1
```

也就是：

- 如果共有 330 只有效转债
- 某因子排第 1，则得 `330` 分
- 某因子排第 2，则得 `329` 分
- ...
- 某因子排最后，则得 `1` 分

## 3. 总分计算

代码里会先分别计算两项得分：

- `dblow_score`
- `curr_iss_amt_score`

然后直接相加：

```text
total_score = dblow_score + curr_iss_amt_score
```

总分越高，说明它在“低双低、低规模”这两个维度上整体越靠前。

## 4. 最终排序方式

最终列表按以下顺序排序：

1. `total_score` 降序
2. `dblow` 升序
3. `price` 升序
4. `curr_iss_amt` 升序

这样做的含义是：

- 先比较三低总分
- 如果总分相同，再优先双低更低的
- 如果双低也相同，再参考价格和规模

## 5. 消息里的展示内容

为了便于人工核对，企业微信消息里会直接展示：

- `三低总分`
- `双低排名/得分`
- `规模排名/得分`

这样看到某只债时，可以直接知道它为什么排在当前位置。

## 6. 防止排序失真

三低排序必须基于全量数据，否则名次会失真。

因此当前实现里做了两层保护：

- 请求参数里的 `rp` 提高到较大的值，尽量一次拉全
- 拉取后校验 `len(rows)` 和接口返回的 `total`

如果未来接口真的改成分页，并且返回的 `rows` 少于 `total`，代码会直接报错，避免在不完整数据上继续做三低排名。

## 7. 对应代码位置

主要逻辑都在 `cb_main.py`：

- 基础过滤：`get_cb_filter_reasons()`、`filter_cb()`
- 数值转换：`to_float()`
- 单项打分计算：`assign_factor_scores()`
- 三低总排序：`sort_cb_rows()`
- 消息展示：`format_cb()`

## 8. 当前结论

当前代码逻辑等价于下面这句话：

> 双低越低越好，规模越低越好；双低与规模得分直接相加，总分越高越靠前。
